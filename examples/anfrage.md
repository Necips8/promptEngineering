# Ursprüngliche Anfrage: Demonstration des Multi-Agenten-Workflows

**Ziel:** Erstellung einer durchgängigen Beispiel-Dokumentation, die zeigt, wie die spezialisierten System-Prompts gemäß dem empfohlenen Workflow (siehe `prompts/README.md`) eingesetzt werden.

**Anforderungen:**
1.  **Szenario:** Definition einer fiktiven Kundenanforderung für ein neues Software-Feature oder Produkt.
2.  **Prozess:** Schrittweise Durchführung aller Phasen des Entwicklungszyklus:
    *   Phase 0: Strategie & Markt-Analyse
    *   Phase 1: Exploration & Definition (inkl. Kunden-Feedback und kritischer Prüfung)
    *   Phase 2: Planung & Architektur
    *   Phase 3: Implementierung & Testing (simuliert durch eine Kernfunktion)
    *   Phase 4: Qualitätssicherung & Release
3.  **Rollen:** Einnahme der jeweils vorgeschlagenen Rolle (z. B. Strategie-Analyst, Architekt, Coder) pro Phase.
4.  **Dokumentation:** Erstellung einer separaten Markdown-Datei für jede Phase im Verzeichnis `examples/`.
5.  **Umfang der Implementierung:** Für Phase 3 ist eine exemplarische Implementierung einer Funktion ausreichend, um den Übergang von Architektur zu Code zu verdeutlichen.

**Zweck:** Diese Beispiele dienen als Referenz und Best-Practice-Leitfaden für die Nutzung des Multi-Agent-Systems.
